# BuildTrack – UI Design

## Designziel

BuildTrack soll wie eine moderne SaaS-Anwendung wirken:

- klar
- aufgeräumt
- hochwertig
- schnell verständlich
- mobil gut bedienbar

## Produktname

BuildTrack

## Untertitel

Baustellen- & Zeiterfassung

## Design-System

Verwendet wird:

- Tailwind CSS
- shadcn/ui
- Nova Preset
- Lucide Icons
- Geist Font

## Layout

Die Anwendung verwendet eine klassische Dashboard-Struktur:

- Sidebar links
- Topbar oben
- Hauptinhalt rechts
- mobile Navigation für kleine Bildschirme

## Navigation

Hauptpunkte:

- Dashboard
- Baustellen
- Mitarbeiter
- Zeiterfassung
- Fotos
- Auswertungen
- Einstellungen

## Farben

### Primär

Blau für Aktionen und wichtige Elemente.

### Hintergrund

Heller, ruhiger Hintergrund für professionelle Lesbarkeit.

### Karten

Weiße oder leicht abgesetzte Flächen mit klaren Abständen.

### Statusfarben

- Grün: aktiv / erfolgreich
- Gelb: Warnung / offen
- Rot: Fehler / abgelehnt
- Grau: inaktiv / archiviert

## UI-Grundsätze

- Große Buttons auf mobilen Geräten
- Wenige Klicks für Mitarbeiter
- Klare Beschriftungen
- Keine überladenen Tabellen
- Wichtige Aktionen gut sichtbar
- Admin-Ansicht darf detaillierter sein
- Mitarbeiter-Ansicht muss extrem einfach sein
